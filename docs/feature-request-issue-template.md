# Feature request issue template

**Source:** [`.github/ISSUE_TEMPLATE/feature_request.yml`](../.github/ISSUE_TEMPLATE/feature_request.yml)

## Purpose

Captures feature ideas with motivation, proposed solution, priority, and optional contribution intent.

## GitHub metadata

| Field | Value |
|-------|--------|
| Form name | Feature Request |
| Default title | `[Feature]: ` (user completes) |
| Labels | `enhancement`, `triage` |

## Form fields

| Field ID | Type | Required | Description |
|----------|------|----------|-------------|
| *(intro)* | markdown | — | Thank-you message |
| `summary` | textarea | yes | Brief feature summary |
| `motivation` | textarea | yes | Problem and use case |
| `solution` | textarea | yes | Proposed behavior |
| `alternatives` | textarea | no | Other options considered |
| `examples` | textarea | no | Usage examples (optional code block) |
| `priority` | dropdown | yes | Low / Medium / High / Critical |
| `contribution` | checkboxes | no | PR, testing, or docs help |
| `terms` | checkboxes | yes | No duplicates, community benefit, sufficient detail |

## Customize after using the template

- Update form `description` to your project name.
- Change `priority` options if you use a different triage scale.
- Remove the `examples` code placeholder language if not applicable.
- Align labels with your issue taxonomy (e.g. `feature` instead of `enhancement`).
